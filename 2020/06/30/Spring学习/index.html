一、概述

二、IOC（控制反转）

控制反转IoC（Inversion of Control）是一种设计思想，DI（依赖注入）是实现IoC的一种方法，控制反转可以理解成获得依赖对象的方式反转了。

我们来回顾一下之前传统的MVC架构，在dao层实现了UserDao接口，再去编写一个对应的接口实现类，之后在service层调用dao层的方法，那么首先就要有这个接口实现类的对象，我们才能够调用里面的具体方法，所以我们new出来了一个接口实现类对象。

// UserDaoImpl
public class UserDaoImpl implements UserDao {
    public void getInfo() {
        System.out.println("我是UserDao的一个实现类：UserDaoImpl");
    }
}
// UserServiceImpl
public class UserServiceImpl implements UserService {
    UserDao userDao = new UserDaoImpl();
    public void getInfo() {
        userDao.getInfo();
    }
}
// 模拟用户
@Test
public void getInfo(){
    // 调用UserDaoImpl中的方法
    UserService userService = new UserServiceImpl();
    userService.getInfo();
}

那么假设现在在接口下面增加了很多个其他的实现类，现在我们的用户需要去调用别的接口实现类，那么在service层new出来的对象就需要重新修改了。这样子的话，用户每次需要调用别的其他的方法，我们就需要去service层修改对应的代码（创建不同的对象）去满足用户的需求。

// UserDaoImplement02
public class UserDaoImplement02 implements UserDao {
    public void getInfo() {
        System.out.println("我是UserDao的一个实现类：UserDaoImplement02");
    }
}

用户仍然可以通过创建一个service对象，让service自己去调用对应的实现类，那么这时候就需要修改service的代码为，

public class UserServiceImpl implements UserService {

    UserDao userDao = new UserDaoImplement02();

    public void getInfo() {
        userDao.getInfo();
    }
}

显然这样的操作无意义，那么我们考虑在service层中增加一个接口的set方法，当用户需要调用什么底层实现的时候，只要把那个实现类丢给service的set方法就可以了，程序员无须进行任何其他操作。

UserDao userDao;

public void setUserDao(UserDao userDao) {
    this.userDao = userDao;
}

用户调用如下，将创建对象的控制权掌握在自己的手里，

@Test
public void getInfo(){
    // 调用UserDaoImpl中的方法
    UserServiceImpl userService = new UserServiceImpl();
    userService.setUserDao(new UserDaoImpl());
    userService.getInfo();
    //现在想要调用UserDaoImplement02中的方法
    userService.setUserDao(new UserDaoImplement02());
    userService.getInfo();
}

控制：就是谁控制依赖对象的创建，传统的程序是由程序本身控制创建，Spring就接管了对象的创建；

反转：程序本身不创建对象，而变成被动的接受对象。

那么在Spring中，对象就是交给Spring容器来创建，Spring本质上还是利用set方法进行注入的（依赖注入）。

三、Spring的IOC实现

1、初识Spring

还是上面那个例子，现在使用spring的IOC操作，将对象的创建完全交给spring，在程序中不存在new了。

先创建一个spring的配置文件，名称可以不固定，这是使用beans.xml，放在resources资源文件夹中。bean元素中的id属性表示唯一标识，class表示需要创建对象的类是谁，使用全限定类名。bean中海油一个property标签，这个代表类中的属性，可以通过name找到对应的属性，使用value为它赋值，由于这里的属性是一个对象，所以使用ref应用当前bean容器中id为userDaoImpl01的bean。说白了，bean就是一个对象。

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="userDaoImpl01" class="com.zhouhaofeng.dao.UserDaoImpl"/>
    <bean id="userDaoImpl02" class="com.zhouhaofeng.dao.UserDaoImplement02"/>

    <bean id="serviceImpl" class="com.zhouhaofeng.service.UserServiceImpl">
        <property name="userDao" ref="userDaoImpl01"/>
    </bean>

</beans>

在测试类中测试，获取的时候需要先获取到上下文对象，就是拿到beans中配置的对象。之后想要创建什么对象，直接使用getBean(“id”)，获取在bean中注册过的对象即可。如果这里想要调用serviceImpl02那个实现类，那么只需要在beans.xml文件中的属性注入的ref值改为想要获取的那个对象就可以了。

@Test
public void getInfo(){
    ClassPathXmlApplicationContext context =
        new ClassPathXmlApplicationContext("beans.xml");
    UserService serviceImpl = (UserService) context.getBean("serviceImpl");
    serviceImpl.getInfo();
}

2、IOC创建对象的方式

IOC创建对象需要类的无参构造，无参构造隐式存在，除非使用有参构造，那么无参构造函数就会被干掉。如果我们给一个类添加有参构造，而没有显式写一个无参构造，那么spring会初始化失败。

如何解决？同样在配置文件中配置传入的参数就可以了。

// User类的有参构造
public User(String name){
    System.out.println("User的有参构造方法");
    this.name = name;
}

在xml中配置，这只是其中一种方式，还有另外两种方式，但是这里不介绍，不常用。

<bean id="user" class="com.zhouhaofeng.pojo.User">
    <constructor-arg name="name" value="周毫锋"/>
</bean>

在配置文件加载的时候，里面的所有bean对象都被创建出来了，不管你用不用。

4、Bean作用域

这里先掌握前面两种，单例模式和原型模式。

Scope Description
singleton (Default) Scopes a single bean definition to a single object instance for each Spring IoC container.
prototype Scopes a single bean definition to any number of object instances.
request Scopes a single bean definition to the lifecycle of a single HTTP request. That is, each HTTP request has its own instance of a bean created off the back of a single bean definition. Only valid in the context of a web-aware Spring ApplicationContext.
session Scopes a single bean definition to the lifecycle of an HTTP Session. Only valid in the context of a web-aware Spring ApplicationContext.
application applicationext`.
websocket Scopes a single bean definition to the lifecycle of a WebSocket. Only valid in the context of a web-aware Spring ApplicationContext.

单例模式：Spring默认的作用域，不过你get多少次，都是原来那个对象。

<bean id="accountService" class="com.something.DefaultAccountService" scope="singleton"/>

原型模式：每次从容器中getBean的时候，都会产生一个新的对象。

<bean id="accountService" class="com.something.DefaultAccountService" scope="prototype"/>

六、Bean的自动装配

1、理解原生自动装配

在前面的配置bean都是自己手动在xml配置文件中进行配置的，现在了解一下spring的bean自动装配机制。

下面的bean创建是手动创建的，

<bean id="person" class="com.zhouhaofeng.pojo.Person">
    <property name="name" value="周毫锋"/>
    <property name="cat" ref="cat"/>
    <property name="dog" ref="dog"/>
</bean>

<bean id="dog" class="com.zhouhaofeng.pojo.Dog" />
<bean id="cat" class="com.zhouhaofeng.pojo.Cat" />

下面是自动装配的，

<bean id="person" class="com.zhouhaofeng.pojo.Person" autowire="byType">
    <property name="name" value="周毫锋"/>
</bean>
--------------------------------------------------------------------------
<bean id="person" class="com.zhouhaofeng.pojo.Person" autowire="byName">
    <property name="name" value="周毫锋"/>
</bean>
<bean id="dog" class="com.zhouhaofeng.pojo.Dog" />
<bean id="cat" class="com.zhouhaofeng.pojo.Cat" />

autowire=byName，那么Spring会在容器上下文中查找，和自己对象set方法后面的属性值对应的bean id。

autowire=byType，那么Spring会在容器上下文中查找，和自己对象set方法后面的属性值对象的类型，这个方法需要保证类型全局唯一，也就是一个类只有一个bean在容器里。

八、使用java方式配置Spring（无xml）

spring可以实现只用java代码实现配置，无须使用对应的配置文件。Spring有一个子项目是JavaConfig，就是做这个的。

带有@Configuration的类是配置类，相当于beans.xml。带有@bean的方法，相当于之前的bean标签。

/**
 * 配置类
 *  - Configuration : 相当于之前xml文件的<beans></beans>标签，可以在里面添加bean
 *  - ComponentScan : 会去扫描对应路径下面的带有component注解的类
 *  - Bean :
 *      - 原来的bean注册：<bean id="user" class="com.zhouhaofeng.pojo.user"></bean>
 *      - 现在是方法：返回值表示类的地址class路径，id表示全局唯一标识，getBean的名字就是这个ID
 * @author zhf
 * @create 2020-05-29 10:44
 */
@Configuration
@ComponentScan("com.zhouhaofeng.pojo")
public class MyConfig {

    @Bean
    public User user(){
        return new User();
    }
}

原来的实体类仍然使用@Component注解注入到bean，使用@Value将属性注入到bean中，

@Component
public class User {
    @Value("周毫锋")
    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "User{" +
                "name='" + name + '\'' +
                '}';
    }
}

在测试类中进行测试，这里没有了xml文件，所以之前获取上下文的方式也就不适用了，这里使用另外一个方式，

@Test
    public void test01(){

        ApplicationContext context = new AnnotationConfigApplicationContext(MyConfig.class);

        System.out.println(user.getName());
    }

九、代理模式

SpringAOP的底层就是代理模式。

1、静态代理

在静态代理中存在一个抽象角色，一般会是接口或者抽象类，可以理解这是一个具体的目标；这里还存在一个真实角色，这个角色想要去实现这个抽象角色，但是这之间可能存在很多复杂的非核心业务逻辑，但是这个真实角色不想去搞这么多复杂的关系；这时候就出现了一个代理角色，这个代理角色会去把实现抽象类之间所有繁琐的操作都实现了。这就是静态代理。

代理类和被代理类要实现同一个主题接口，而且代理类中要有一个被代理类的属性，这样才能把核心业务逻辑交还给被代理类完成；而一些与核心业务逻辑无关的逻辑，并且需求是多变的，那么这些逻辑就可以交给代理类来完成。

一般使用静态代理给系统增加某些功能，假设在下面的例子中，抽象角色就是下面的UserService，

public interface UserService {
    void addUser();
    void deleteUser();
}

通常我们会有UserServiceImpl去实现这个接口，UserServiceImpl就是真实角色，

public class UserServiceImpl implements UserService {
    public void addUser() {
        System.out.println("UserServiceImpl--> addUser");
    }

    public void deleteUser() {
        System.out.println("UserServiceImpl--> deleteUser");
    }
}

好了，这时候如果多了一个其他需求，在实现接口的时候要多输出一句日志，在基础的方法就是在真实角色里面修改代码，加上一句对应的syso就可以了；但是现在不想修改源代码，让UserServiceImpl专注做他的业务逻辑，增加的需求使用代理角色完成，

public class UserServiceProxy implements UserService {

    UserServiceImpl userServiceImpl;

    public void setUserServiceImpl(UserServiceImpl userServiceImpl) {
        this.userServiceImpl = userServiceImpl;
    }

    public void addUser() {
        userServiceImpl.addUser();
        System.out.println("UserServiceProxy--> addUser");
    }

    public void deleteUser() {
        userServiceImpl.deleteUser();
        System.out.println("UserServiceProxy--> addUser");
    }
}

这时候就可以完成需求了，而不需要在UserServiceImpl中去增加代码，虽然代码量是多了一点，但是确实是一个好的解决方法。记得在代理对象中需要持有真实角色的引用，就是把人家注入进来，这里是set方法注入。

2、动态代理

使用静态代理只能为一个类服务，如果需要代理的类很多，那么就需要编写大量的代理类，比较繁琐。

使用动态代理，接口、被代理类不变，我们构建一个handler类来实现InvocationHandler接口。

2.1、创建代理的调用处理器

这个处理器相当于就是创建动态代理类的一个模板，通过这个我们可以得到我们的代理类。

这个类需要实现 InvocationHandler 接口，实现接口中的invoke方法，生成获取创建动态代理类的方法。

在静态代理中我们知道，代理类需要真实角色（被代理对象）的引用，同时需要实现和被代理对象一样的接口。使用首先要注入一个对应的被代理对象；接下来想要得到一个代理类，还需要实现同一个接口，所以在getProxy中，使用newProxyInstance获取一个动态代理类，这个方法需要被代理角色，抽象角色，以及调用处理器；最后的invoke方法，是用来执行被代理对象中的方法的。

public class ProxyHandler implements InvocationHandler {

    // 被代理对象（真实角色）
    private Object object;

    // 注入被代理对象，方便调用方法并返回结果
    public void setObject(Object object) {
        this.object = object;
    }

    // 自动获取代理类
    public Object getProxy(){
        return Proxy.newProxyInstance(this.object.getClass().getClassLoader(),
                this.object.getClass().getInterfaces(), this);
    }

    // 执行代理对象中的方法，并可以增加一些其他的方法
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("使用动态代理模式前");
        method.invoke(object, args);
        System.out.println("使用动态代理模式后");
        return null;
    }
}

测试如下：

@Test
public void testProxy(){
    // 创建一个被代理对象
    UserServiceImpl userServiceImpl = new UserServiceImpl();
    // 创建代理调用处理器
    ProxyHandler proxyHandler = new ProxyHandler();
    // 注入被代理对象
    proxyHandler.setObject(userServiceImpl);
    // 获取动态代理类
    UserService proxy = (UserService) proxyHandler.getProxy();
    // 执行方法
    proxy.sayHello();
}

增加一个日志增强效果，就是在方法UserService前和后输出一写日志，跟代理的思路一样，

// 方法执行之前
public class BeforeLog implements MethodBeforeAdvice {
    /**
     *
     * @param method 要执行的目标对象的方法
     * @param args 方法的参数
     * @param target 目标对象
     * @throws Throwable
     */
    public void before(Method method, Object[] args, Object target) {
        System.out.println(target.getClass().getName()+"的"+method.getName()+"被执行了....");
    }
}

// 取到方法返回值之后
public class AfterLog implements AfterReturningAdvice {
    public void afterReturning(Object returnValue, Method method,
                               Object[] args, Object target) throws Throwable {

        System.out.println("执行了"+method.getName()+"方法，返回结果为："+returnValue);
    }
}

在xml文件中对aop进行配置，需要引入aop约束

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:aop="http://www.springframework.org/schema/aop"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd
       http://www.springframework.org/schema/aop
       http://www.springframework.org/schema/aop/spring-aop.xsd">

    <bean id="userService" class="com.zhouhaofeng.service.UserServiceImpl"/>
    <bean id="afterLog" class="com.zhouhaofeng.log.AfterLog" />
    <bean id="beforeLog" class="com.zhouhaofeng.log.BeforeLog" />

    <!--配置aop-->
    <aop:config>
        <!--切入点，要执行的位置：execution()-->
        <aop:pointcut id="pointcut" expression="execution(* com.zhouhaofeng.service.UserServiceImpl.*(..))"/>
        <aop:advisor advice-ref="afterLog" pointcut-ref="pointcut"/>
        <aop:advisor advice-ref="beforeLog" pointcut-ref="pointcut" />
    </aop:config>

</beans>

对结果进行测试：

@Test
public void test01(){
    ClassPathXmlApplicationContext context =
            new ClassPathXmlApplicationContext("applicationContext.xml");
    UserService userService = (UserService) context.getBean("userService");
    userService.addUser();
}

2、自定义切面实现aop

现在自定义一个方法，这个方法就是代理对象增加的方法，假设我们现在给UserServiceImpl的方法增加一个方法执行之前和之后的日志输出，那么先将这些方法定义到一个类中，

public class DiyMethod {
    public void before(){
        System.out.println("==========方法执行之前==========");
    }

    public void after(){
        System.out.println("==========方法执行之前==========");
    }
}

之后在xml文件中进行配置，先将自定义注入到bean容器中，随后进行aop配置。

在aop配置的时候，使用切面配置，这个切面可以理解为就是增加的类，在切面中定义切入点，就是需要将自定义的方法添加到什么位置，之后进行通知，添加before和after两个操作。

<bean id="diyMethod" class="com.zhouhaofeng.diy.DiyMethod" />
<aop:config>
    <!--定义切面-->
    <aop:aspect ref="diyMethod">
        <!--定义切入点-->
        <aop:pointcut id="point" expression="execution(* com.zhouhaofeng.service.UserServiceImpl.*(..))"/>
        <!--通知-->
        <aop:before method="before" pointcut-ref="point"/>
        <aop:after method="after" pointcut-ref="point"/>
    </aop:aspect>
</aop:config>

测试如下：

@Test
    public void test02(){
        ApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml");
        UserService userService = (UserService) context.getBean("userService");
        userService.addUser();
    }

效果如下：这个方式与代理方式一样，容易理解。

3、注解实现aop

使用注解实现aop较为简单，@Aspect将一个类声明为切面，@Before表示在方法执行之前的操作，需要添加对应的切入点。@Before是前置增强，@After是后置增强，@Around是环绕增强，关注一下执行顺序。

@Aspect
public class AnnotationProxy {
    @Before("execution(* com.zhouhaofeng.service.UserServiceImpl.*(..))")
    public void before(){
        System.out.println("AnnotationProxy==========方法执行之前==========");
    }

    @After("execution(* com.zhouhaofeng.service.UserServiceImpl.*(..))")
    public void after(){
        System.out.println("AnnotationProxy==========方法执行之前==========");
    }

    @Around("execution(* com.zhouhaofeng.service.UserServiceImpl.*(..))")
    public void around(ProceedingJoinPoint joinPoint) throws Throwable {
        System.out.println("环绕前");
        joinPoint.proceed(); //执行方法
        System.out.println("环绕后");
    }

}

编写完成类之后，在xml中配置开启注解支持就可以了。

<bean id="anno" class="com.zhouhaofeng.diy.AnnotationProxy"/>
    <aop:aspectj-autoproxy/>

结果为，==关注顺序==

3、配置spring整合的xml文件

在resources文件夹下面新建一个spring-dao.xml文件，用来配置于数据库相关的操作。首先配置数据源，使用spring里面的DataSource进行配置，然后可以去除掉mybatis中的数据源配置（原来在mybatis-config.xml）。

接下来配置sqlSessionFactory，然后注入数据源，绑定mybatis的核心配置文件，然后在这里也可以注册mapper，或者修改别名等等操作，其实可以不用绑定核心配置文件，将核心配置文件中的所有操作都在这里注入就可以了。

接下来配置SQLSession，需要注入sqlSessionFactory，可以看一下源码，它没有set方法，所以只能选择构造器注入。spring里面的是SqlSessionTemplate，一般我们仍然还是用mybatis中的SqlSession，方便理解。

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--dataSource，替换掉原来mybatis的数据源，引入spring-jdbc-->
    <bean id="dataSource" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
        <property name="driverClassName" value="com.mysql.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://localhost:3306/mybatis?useUnicode=true&amp;characterEncoding=UTF-8"/>
        <property name="username" value="root"/>
        <property name="password" value="123456"/>
    </bean>

    <!--sqlSessionFactory-->
    <bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
        <property name="dataSource" ref="dataSource"/>
        <!--绑定mybatis配置文件，可以在这里配置mybatis的属性，那么可以省去xml配置文件-->
        <property name="configLocation" value="classpath:mybatis-config.xml"/>
        <!--注册所有的mapper-->
        <property name="mapperLocations" value="classpath*:com/zhouhaofeng/mapper/*Mapper.xml"/>
    </bean>

    <!--创建SQLSession的对象，用构造器注入sqlSessionFactory-->
    <bean id="sqlSession" class="org.mybatis.spring.SqlSessionTemplate">
        <constructor-arg name="sqlSessionFactory" ref="sqlSessionFactory"/>
    </bean>

</beans>

在上面使用到了classpath路径，这个路径是指使用maven打包后的文件资源中的classes文件夹。

4、新建一个UserMapperImpl

新建了一个接口的实现类UserMapperImpl，原来我们是在测试的时候去拿到接口的mapper，然后调用方法，这里是将这些方法在实现类中搞定，测试类直接调用实现类就可以了。

public class UserMapperImpl implements UserMapper {

    //注入sqlSession
    private SqlSessionTemplate sqlSession;

    public void setSqlSession(SqlSessionTemplate sqlSession) {
        this.sqlSession = sqlSession;
    }

    public List<User> getUserList() {
        UserMapper mapper = sqlSession.getMapper(UserMapper.class);
        return mapper.getUserList();
    }
}

如果这里不想显示声明一个SqlSessionTemplate属性的话，也可以不用，只要继承了SqlSessionDaoSupport类就可以了。

public class UserMapperImpl extends SqlSessionDaoSupport implements UserMapper {
    public List<User> getUserList() {
        return getSqlSession().getMapper(UserMapper.class).getUserList();
    }
}

注入的时候需要注意一下，注入sqlSessionFactory

<!--userMapperImpl-->
<bean id="userMapper" class="com.zhouhaofeng.dao.UserMapperImpl">
    <property name="sqlSessionFactory" ref="sqlSessionFactory"/>
</bean>

5、注册实现类到bean容器

这时候为了方便，将spring-dao.xml的操作都是固定的，所以抽取出来，然后将我们的实现类的bean注册到applicationContext.xml中，导入spring-dao的配置即可。

所以在这里UserMapper是抽象接口，UserMapper.xml是SQL语句的书写，UserMapperImpl是抽象接口的实现类。需要调用的获取UserMapperImpl的bean就可以了。

    </bean>

</beans>

十二、声明式事务

1、事务

==要么都成功，要么都失败。==

ACID原则：

  • 原子性（Atomicity）
    原子性是指事务是一个不可分割的工作单位，事务中的操作要么都发生，要么都不发生。
  • 一致性（Consistency）
    事务前后数据的完整性必须保持一致。
  • 隔离性（Isolation）
    事务的隔离性是多个用户并发访问数据库时，数据库为每一个用户开启的事务，不能被其他事务的操作数据所干扰，多个并发事务之间要相互隔离。
  • 持久性（Durability）
    持久性是指一个事务一旦被提交，它对数据库中数据的改变就是永久性的，接下来即使数据库发生故障也不应该对其有任何影响。

2、配置声明式事务

这里增加了addUser和deleteUser两个 方法，故意在delete的SQL语句中写了错误

public class UserMapperImpl implements UserMapper {

    private SqlSessionTemplate sqlSession;

    public void setSqlSession(SqlSessionTemplate sqlSession) {
        this.sqlSession = sqlSession;
    }

    public List<User> getUserList() {
        User user = new User(8, "小雨", "457896");
        sqlSession.getMapper(UserMapper.class).addUser(user);
        sqlSession.getMapper(UserMapper.class).deleteUser(7);
        return sqlSession.getMapper(UserMapper.class).getUserList();
    }

    public int addUser(User user) {
        return sqlSession.getMapper(UserMapper.class).addUser(user);
    }

    public int deleteUser(int id) {
        return sqlSession.getMapper(UserMapper.class).deleteUser(id);
    }
}

那么我们期望的是删除错误之后，添加的数据会被回滚，不会添加到数据库中去，所以下面配置spring声明式事务。

要开启 Spring 的事务处理功能，在 Spring 的配置文件中创建一个 DataSourceTransactionManager 对象：

<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
  <constructor-arg ref="dataSource" />
</bean>

在spring-dao.xml中配置

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:tx="http://www.springframework.org/schema/tx"
       xmlns:aop="http://www.springframework.org/schema/aop"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/aop
        https://www.springframework.org/schema/aop/spring-aop.xsd
        http://www.springframework.org/schema/tx
        https://www.springframework.org/schema/tx/spring-tx.xsd">

    <bean id="dataSource" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
        <property name="driverClassName" value="com.mysql.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://localhost:3306/mybatis?useUnicode=true&amp;characterEncoding=UTF-8"/>
        <property name="username" value="root"/>
        <property name="password" value="123456"/>
    </bean>

    <bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
        <property name="dataSource" ref="dataSource"/>
        <property name="configLocation" value="classpath:mybatis-config.xml"/>
        <property name="mapperLocations" value="classpath*:com/zhouhaofeng/mapper/*Mapper.xml"/>
    </bean>

    <bean id="sqlSession" class="org.mybatis.spring.SqlSessionTemplate">
        <constructor-arg name="sqlSessionFactory" ref="sqlSessionFactory"/>
    </bean>

    <!--配置声明式事务-->
    <bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
        <constructor-arg ref="dataSource" />
    </bean>

    <!--结合aop实现事务植入-->
    <tx:advice id="txAdvice" transaction-manager="transactionManager">
        <!--给所有的方法配置事务-->
        <tx:attributes>
            <tx:method name="*" propagation="REQUIRED"/>
        </tx:attributes>
    </tx:advice>

    <!--配置事务切入-->
    <aop:config>
        <aop:pointcut id="txPointCut" expression="execution(* com.zhouhaofeng.mapper.*.*(..))"/>
        <aop:advisor advice-ref="txAdvice" pointcut-ref="txPointCut"/>
    </aop:config>

</beans>

测试之后增加和删除就是同一个事务，要么一起成功，要么一起失败。
