二、SpringMVC执行流程

SpringMVC使用的DispatcherServlet作为中心控制器，将请求分发到不同的处理器。SpringMVC像许多框架一样，以请求为驱动，围绕一个中心Servlet分派请求以及提供其他功能。DispatcherServlet也是一个Servlet，继承自HttpServlet.

在web.xml中配置注册DispatcherServlet，DispatcherServlet需要去找到对应的spring的配置文件

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">

    <!--注册DispatcherServlet-->
    <servlet>
        <servlet-name>springmvc</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <init-param>
            <!--上下文配置的位置的制定-->
            <param-name>contextConfigLocation</param-name>
            <!--此时是在类路径下面去寻找，也就是resources/-->
            <param-value>classpath:springmvc-servlet.xml</param-value>
        </init-param>
        <!--启动级别：1，跟服务器一起启动-->
        <load-on-startup>1</load-on-startup>
    </servlet>
    <!--servlet映射-->
    <servlet-mapping>
        <servlet-name>springmvc</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>

</web-app>

在映射的时候，url-pattern中使用/ 和 /* 的区别是：

  • / ：会去匹配所有的请求，但是不包括jsp页面

  • /* ：会去匹配所有的请求，包括jsp页面

在springmvc-servlet.xml，其实这个配置文件也是spring的配置文件，只不过名字可以随便取而已

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:aop="http://www.springframework.org/schema/aop"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
    http://www.springframework.org/schema/beans/spring-beans-3.0.xsd
    http://www.springframework.org/schema/aop
    http://www.springframework.org/schema/aop/spring-aop-3.0.xsd
    http://www.springframework.org/schema/context
    http://www.springframework.org/schema/context/spring-context.xsd">

    <!--配置Handler解析器-->
    <bean class="org.springframework.web.servlet.handler.BeanNameUrlHandlerMapping"/>
    <!--配置Handler适配器-->
    <bean class="org.springframework.web.servlet.mvc.SimpleControllerHandlerAdapter"/>

    <!--
        配置一个视图解析器
        常用内部资源视图解析器
        这里面的意思是：加入此时viewName为hello,那么它会到WEB-INF/jsp/hello.jsp中去寻找
    -->
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <!--1. 前缀-->
        <property name="prefix" value="/WEB-INF/jsp/"/>
        <!--2. 后缀-->
        <property name="suffix" value=".jsp"/>
    </bean>

    <!--注册controller，这里的id需要加上/，因为在浏览器发起的就是/hello请求-->
    <bean id="/hello" class="com.zhouhaofeng.controller.HelloController"/>

</beans>

在实际控制层controller中，调用业务逻辑，返回模型数据视图

public class HelloController implements Controller {

    public ModelAndView handleRequest(HttpServletRequest Request, HttpServletResponse Response) throws Exception {
        // 模型和视图
        ModelAndView mv = new ModelAndView();
        // 封装对象，放在Modelandview中
        mv.addObject("msg", "hello,SpringMVC!");
        // 跳转到页面
        mv.setViewName("hello");
        return mv;
    }
}

如果项目代码都没有错误，但是仍然报了404错误，那么很有可能是jar包没有添加进来，因为这里使用的是创建一个空的项目然后再去添加框架支持，所以lib文件夹很可能没有。

SpringMVC的执行过程如下：

  1. 用户在浏览器发起请求，DispatcherServlet拦截请求
  2. DispatcherServlet调用HandlerMapping，HandlerMapping为处理器映射，将根据url查找对应的Handler
  3. HandlerExecution根据url查找控制器（controller层）
  4. HandlerExecution将解析后的信息传递给DispatcherServlet，如解析控制器映射
  5. HandlerAdapter按照特定的规则去执行Handler
  6. Handler执行具体的controller
  7. Controller将具体的执行信息返回，如ModelAndView
  8. HandlerAdapter将模型数据或者视图逻辑名返回给DIspatcherServlet
  9. DispatcherServlet调用视图解析器ViewResolver解析逻辑视图名
  10. 视图解析器将解析完成之后的逻辑视图名传给DispatcherServlet
  11. DispatcherServlet根据视图名调用具体的页面
  12. 将页面返回给用户

这一步是固定不变的，配置完成之后就可以不用理会他了。

<?xml version="1.0" encoding="UTF-8"?>

         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
    <servlet>
        <servlet-name>springmvc</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>

    <servlet-mapping>

2、配置springmvc-servlet.xml

这一步是固定不变的，配置完成之后就可以不用理会他了。

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:mvc="http://www.springframework.org/schema/mvc"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
    http://www.springframework.org/schema/beans/spring-beans-3.0.xsd
    http://www.springframework.org/schema/mvc
    http://www.springframework.org/schema/mvc/spring-mvc.xsd
    http://www.springframework.org/schema/context
    http://www.springframework.org/schema/context/spring-context.xsd">

    <!--自动扫描包，让指定包下面的所有注解生效，让IOC容器统一管理-->
    <context:component-scan base-package="com.zhouhaofeng.controller"/>
    <!--过滤静态资源-->
    <mvc:default-servlet-handler />
    <!--注解需要，自动映射-->
    <mvc:annotation-driven />

    <!--视图解析器-->
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver" name="internalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/jsp/"/>
        <property name="suffix" value=".jsp"/>
    </bean>
</beans>

3、写controller

@Controller帮我们干了很多的事情，注册到spring的IOC容器中了，可以直接使用了

@RequestMapping(“/hello”) 表示浏览器的访问路径为/hello

一个方法就相当于之前的一个Servlet，这里的返回值就是视图逻辑名字，会被视图解析器识别加上完成前后缀。

@Controller
@RequestMapping("/hello")
public class HelloController {
    @RequestMapping("/h1")
    public String hello(Model model){
        model.addAttribute("msg", "你好，这是我使用注解实现的springmvc");
        return "hello"; //视图解析器会拿到这个字符串，加上前后缀
    }
}

四、controller详解

控制器controller提供访问引用程序的行为，通常可以使用接口定义或者注解定义两种方法实现；控制器负责解析用户的请求，并将其转化为一个模型；在springMVC中一个控制器类可以包含多个方法，一个方法相当于之前的一个servlet。

1、通过接口定义实现

创建一个类并且继承Controller

public class HelloController implements Controller {

    public ModelAndView handleRequest(HttpServletRequest Request, HttpServletResponse Response) throws Exception {
        // 模型和视图
        ModelAndView mv = new ModelAndView();
        // 封装对象，放在Modelandview中
        mv.addObject("msg", "hello,SpringMVC!");
        // 跳转到页面
        mv.setViewName("hello");
        return mv;
    }
}

在springmvc-servlet.xml中还需要配置这个controller

<!--注册controller，这里的id需要加上/，因为在浏览器发起的就是/hello请求-->
    <bean id="/hello" class="com.zhouhaofeng.controller.HelloController"/>

这种方式一个类只能实现一个方法，同时还要去注册bean，相对麻烦，一般不用。

2、使用注解定义实现

使用@Controller表明了这个类是控制器，相同的还有@Component，@Service，@Repository，会被spring接管。

@Controller
@RequestMapping("/hello")
public class HelloController {
    @RequestMapping("/h1")
    public String hello(Model model){
        model.addAttribute("msg", "你好，这是我使用注解实现的springmvc");
        return "hello"; //视图解析器会拿到这个字符串，加上前后缀
    }
}

方法的返回值

如果类上面标注了@Controller并且方法上面没有标注@ResponseBody的话，return的值就会被视图解析器处理；

如果类上标注的是@RestController的话，或者在方法上使用@ResponseBody，返回值就不会被视图解析器处理

3、@RequestMapping

将url映射到控制器类，可以用在方法上也可以用在类上面。

五、RestFul风格

1、区别

Restful就是一个资源定位和资源操作的风格，传统的url可能会在后面使用问号携带参数，但是restful风格就不会有这些参数带上去，url很简洁。传统的方式只有get和post两种方式，restful操作资源可以通过不同的请求方式实现不同的效果。请求地址可以一样，但是功能可以不一样。

http://localhost:8080/item/1 查询，GET

http://localhost:8080/item 新增，POST

http://localhost:8080/item 更新，PUT

http://localhost:8080/item/1 删除，DELETE

可以在下面的例子看出区别，我们编写一个方法实现计数的功能，

@Controller
public class RestFulController {
    @RequestMapping("/add")
    public String add(int a, int b, Model model){
        int result = a + b;
        model.addAttribute("msg", "计算结果为："+result);
        return "hello";
    }
}

那么访问路径必须携带参数： http://localhost:8080/add?a=5&b=10 ，这样子才可以把参数拿到。

但是是一个restful风格的设计，需要使用一个@PathVariable注解，这个注解表示方法里面的参数可以从路径里面获取，需要注意的是这个方法的名称必须和路径上面使用括号的名称一模一样才可以使用。

@Controller
public class RestFulController {
    @RequestMapping("/add/{a}/{b}")
    public String add(@PathVariable int a, @PathVariable int b, Model model){
        int result = a + b;
        model.addAttribute("msg", "计算结果为："+result);
        return "hello";
    }
}

访问路径如下：http://localhost:8080/add/5/10

2、请求方法

可以使用下面的方式配置请求的方法。

@RequestMapping(value = "/add/{a}/{b}", method = RequestMethod.GET)

因此衍生了其他的注解，限制了请求的方式

@GetMapping("/add/{a}/{b}")   //get方法请求
@PostMapping("/add/{a}/{b}")  //post方法请求
@DeleteMapping("/add/{a}/{b}")  //post方法请求

3、重定向和转发

在springMVC中使用视图解析器，在方法中使用return返回的字符串就是使用转发的方式，如果使用重定向，需要在返回值前面加上redirect。也可以重定向到另外一个请求上面，就不加jsp后缀了。

@RequestMapping("/dispatcher")
public String dispatcher(HttpServletRequest request){
    System.out.println("session为："+request.getSession().getId());
    return "redirect:/index.jsp";
}

4、从前端接受参数

从前端接受的参数，全部使用@RequestParam(“name”)获取到对应的值，这个跟@PathVariable不太一样，后者需要在映射路径中用{}占位，但是前者只要前端传来了一个相同name的值，后端就可以获取到这个值。

@Controller
public class ParamsController {
    @RequestMapping("/user/name")
    public String getUserName(@RequestParam("name")String name, Model model){
        model.addAttribute("msg", name);
        return "success";
    }
}

在浏览器访问：http://localhost:8080/user/name?name=zhouhaofeng 即可。

如果前端接受到的是一个对象呢？

那么在方法中直接使用一个User对象接受就可以了，会自动匹配User中的属性名，如果一样就把值拿过来，不一致就直接为null。

@GetMapping("/user")
public String getUser(User user) {
    System.out.println(user);
    return "success";
}

在浏览器访问：http://localhost:8080/user?name=zhouhaofeng&id=1 即可，注意访问的时候顺序可以不一样，反正只要跟User里面的属性名一致就可以了，如果写的参数多了的话，比如在后面多了一个&age=19，那么在这个多余的参数就不会被获取到了。

5、中文乱码问题

从前端接受的参数如果存在中文，那么在controller中接受到中文的时候就已经是乱码了，但是get方法请求就不会产生中文乱码问题，Tomcat给解决了。

5.1、传统的解决方式

使用过滤器进行拦截处理，先写过滤器

public class EncodingFilter implements Filter {
    public void init(FilterConfig filterConfig) throws ServletException {

    }

    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain) throws IOException, ServletException {
        servletRequest.setCharacterEncoding("utf-8");
        servletResponse.setCharacterEncoding("utf-8");
        filterChain.doFilter(servletRequest, servletResponse);
    }

    public void destroy() {

    }
}

然后在web.xml中进行配置，注意路径还是/*

<!--注册拦截器-->
<filter>
    <filter-name>encoding</filter-name>
    <filter-class>com.zhouhaofeng.filter.EncodingFilter</filter-class>
</filter>
<filter-mapping>
    <filter-name>encoding</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

这样子也可以解决中文乱码，但是重新写一个filter太麻烦了。

5.2、spring解决乱码问题

spring也帮我们解决了中文乱码问题，在实现上其实等同于第一种，只不过我们不用去自己写对应的过滤类了。

<!--配置springMVC的乱码问题解决-->
    <filter>
        <filter-name>encoding</filter-name>
        <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
        <init-param>
            <param-name>encoding</param-name>
            <param-value>utf-8</param-value>
        </init-param>
    </filter>
    <filter-mapping>
        <filter-name>encoding</filter-name>
        <url-pattern>/</url-pattern>
    </filter-mapping>

六、JSON

1、什么是JSON？

JavaScript Object Notation是一种轻量级的数据交换格式，采用完全独立于编程语言的文本格式来存储数据，便于前后端传输数据。

JSON采用键值对形式，形如下面：

{"name" : "zhouhaofeng"}
{"age" : 3}
{"sex" : "男"}

2、Jackson使用

使用Jackson需要先导入依赖

<!-- https://mvnrepository.com/artifact/com.fasterxml.jackson.core/jackson-databind -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.11.0</version>
</dependency>

解决json格式的乱码问题：在springmvc-servlet.xml中进行配置

<!--json乱码问题-->
<mvc:annotation-driven>
    <mvc:message-converters>
        <bean class="org.springframework.http.converter.StringHttpMessageConverter">
            <constructor-arg value="UTF-8"/>
        </bean>
        <bean class="org.springframework.http.converter.json.MappingJackson2HttpMessageConverter">
            <property name="objectMapper">
                <bean class="org.springframework.http.converter.json.Jackson2ObjectMapperFactoryBean">
                    <property name="failOnEmptyBeans" value="false"/>
                </bean>
            </property>
        </bean>
    </mvc:message-converters>
</mvc:annotation-driven>

接下来在controller中实现将任何数据类型转化成json格式返回，这里使用的是@Controller和@ResponseBody配置，跳过视图解析器，直接将结果返回到页面；也可以在类上面标砖为@RestController，效果一样。

需要先声明一个ObjectMapper对象，然后使用writeValueAsString方法返回string。

@Controller
public class UserController {
    @RequestMapping("/user")
    @ResponseBody
    public String getJSON() throws JsonProcessingException {
        User user = new User(1, "周毫锋", 13);
        ObjectMapper mapper = new ObjectMapper();
        // 转换成json格式
        String s = mapper.writeValueAsString(user);
        return s;
    }

    @RequestMapping("/getUser")
    @ResponseBody
    public String getListUser() throws JsonProcessingException {
        List<User> userList = new ArrayList<>();
        User user = new User(1, "周毫锋", 13);
        User user1 = new User(2, "周毫锋", 15);
        User user2 = new User(3, "周毫锋", 14);
        User user3 = new User(3, "周毫锋", 16);
        User user4 = new User(1, "周毫锋", 17);
        userList.add(user);
        userList.add(user1);
        userList.add(user2);
        userList.add(user3);
        userList.add(user4);
        ObjectMapper objectMapper = new ObjectMapper();
        String s = objectMapper.writeValueAsString(userList);
        return s;
    }

    @RequestMapping("/getDate")
    @ResponseBody
    public String getDate() throws JsonProcessingException {
        Date date = new Date();
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        return new ObjectMapper().writeValueAsString(sdf.format(date));
    }
}

由于在上面的操作中存在过多的重复操作，所以抽取成一个工具类，获取json的方法；这里使用了方法的重载，很多源码都是这么做的。

/**
 * 抽取json获取方法，使用方法重载，实现代码复用
 * @author zhf
 * @create 2020-05-31 21:02
 */
public class JsonUtils {

    public static String getJson(Object obj) {
        return getJson(obj, "yyyy-MM-dd HH:mm:ss");
    }

    public static String getJson(Object obj, String dateFormat){
        ObjectMapper objectMapper = new ObjectMapper();
        // 不适用时间戳的方式
        objectMapper.configure(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS, false);
        // 自定义时间格式
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        // 将格式设置到ObjectMapper中
        objectMapper.setDateFormat(sdf);
        // 将数据转化为json格式，并返回
        try {
            return objectMapper.writeValueAsString(obj);
        } catch (JsonProcessingException e) {
            e.printStackTrace();
        }
        // 如果什么都没有做，返回null
        return null;
    }
}

那么controller里面就可以直接调用了 。

@Controller
public class UserController {
    @RequestMapping("/user")
    @ResponseBody
    public String getJSON() throws JsonProcessingException {
        return JsonUtils.getJson(new User(1, "周毫锋", 13));
    }

    @RequestMapping("/getUser")
    @ResponseBody
    public String getListUser() throws JsonProcessingException {
        List<User> userList = new ArrayList<>();
        User user = new User(1, "周毫锋", 13);
        User user1 = new User(2, "周毫锋", 15);
        User user2 = new User(3, "周毫锋", 14);
        User user3 = new User(3, "周毫锋", 16);
        User user4 = new User(1, "周毫锋", 17);
        userList.add(user);
        userList.add(user1);
        userList.add(user2);
        userList.add(user3);
        userList.add(user4);
        return JsonUtils.getJson(userList);
    }

    @RequestMapping("/getDate")
    @ResponseBody
    public String getDate() throws JsonProcessingException {
        return JsonUtils.getJson(new Date(), "yyyy-MM-dd HH:mm:ss");
    }
}

七、FastJSON

fastjson的作用跟Jackson，使用方式相对简单。

导入依赖：

<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>fastjson</artifactId>
    <version>1.2.62</version>
</dependency>

直接使用：

@RequestMapping("/fastjson")
@ResponseBody
public String getFastJosn(){
    List<User> userList = new ArrayList<>();
    User user = new User(1, "周毫锋", 13);
    User user1 = new User(2, "周毫锋", 15);
    User user2 = new User(3, "周毫锋", 14);
    User user3 = new User(3, "周毫锋", 16);
    User user4 = new User(1, "周毫锋", 17);
    userList.add(user);
    userList.add(user1);
    userList.add(user2);
    userList.add(user3);
    userList.add(user4);

    return JSON.toJSONString(userList);
}
